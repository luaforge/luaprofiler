Overview

LuaProfiler is a time profiler designed to help finding bottlenecks on your Lua program. If you require LuaProfiler into your program, it will generate a log of all your function calls and their respective running times. You can use this log to generate a summary of the functions you program uses, along with how long it stayed in each of them, or you can import the log into a spreadsheet to analyze it further.

LuaProfiler is free software and uses the same licenseas Lua 5.0.

Current version

The beta version of LuaProfiler version 2.0.0 (for Lua 3.2, 4.0.1 and 5.0.2) is available for download. It has updated documentation and modified log file format. The final version 2.0.0 will have the support for profiling coroutines.

Download

LuaProfiler can be downloaded (source, and binaries for Win32 and MacOS X) from the LuaForge page.

Installation

LuaProfiler can be built from the scratch by configuring and running the makefile or can be used on Win32 by downloading the binaries.

For older versions of Lua (3.2 and 4.0), the profiler must be embedded in your application (linked with it). You must call the function to start profiling from C code. With Lua 5.0 and the support for dynamic library loading, nothing must be done on the original C sources of your application: just put the library on the library path and require it. For details please check the manual.

In order to analyze the raw profile data produced by a profiling session, you must pass the profile data to the summary.lua script included in the distribution. The raw profile data can also be analyzed by a spreadsheet.
