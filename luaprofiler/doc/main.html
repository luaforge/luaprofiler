Overview

LuaProfiler is a time profiler designed to help finding bottlenecks on your Lua program.

LuaProfiler is free software and uses the same license as Lua 5.0.

Current version

LuaProfiler version 1.1 beta is available for download. It now includes support to Lua 5.0.

Download

LuaProfiler can be downloaded (source and binaries) from the following link:

LuaProfiler at luaforge.net

What's new

  • [jul 16 2004] Version 1.1 beta released (support for Lua 5.0)
  • [mar 04 2002] Version 1.0 released (supporting Lua 3.2, Lua NG and Lua 4.0)

Installation

LuaProfiler can be built from the scratch by configuring and running the makefile or can be used by downloading any of the binaries.

For older versions of Lua (3.2 and 4.0), the profiler must be integrated on the main C source code in order to open the profiler lib. With Lua 5.0 and the support for dynamic library loading, nothing must be done on the original C sources of your application: just put the library on the library path and execute the simple lua script to load it before your application loads its first lua file. For details please take a look at the file "bin/lua50" in the source distribution.

In order to analyze the raw profile data produced by a profiling session, lua scripts present on the source distribution are provided under the directory "analyzer" -- the raw profile data can also be analyzed by a spreadsheet. The output from the analyzer generates a GDL file for use with the aiSee program -- see below.

References

LuaProfiler takes great advantage from the integration with aiSee -- a graph visualization tool. It is recommended that you download aiSee and use it with the LuaProfiler.

For instructions on how to use aiSee please refer to "analyzer.txt" present on the source distribution.
